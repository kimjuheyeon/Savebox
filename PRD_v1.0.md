2. 제품 개요
2.1 제품 소개
SaveBox는 인스타그램, 유튜브, 트위터 등 여러 SNS에 흩어진 북마크를 하나의 앱에서 통합 관리할 수 있는 모바일 서비스입니다. SNS의 공유 기능을 활용하여 3초 안에 콘텐츠를 저장하고, 나만의 컬렉션으로 체계적으로 정리할 수 있습니다.
2.2 핵심 가치
•	빠른 저장: SNS 탐색 중 3초 이내 북마크 저장
•	체계적 정리: 컬렉션과 컬러 태그로 직관적 분류
•	쉬운 재발견: 앨범형/리스트형 뷰와 검색으로 빠른 접근
•	멀티 플랫폼: 모든 주요 SNS 지원
2.3 차별화 포인트
•	네이티브 공유 기능 연동으로 앱 전환 없이 즉시 저장
•	사용자 맥락에 따른 이중 UX 구조 (빠른 저장 vs 천천히 탐색)
•	원티드 디자인시스템으로 일관된 개인화 경험
•	앨범형/리스트형 뷰 전환으로 다양한 탐색 방식 지원



3. 목표 및 배경
3.1 문제 정의
•	각 SNS마다 북마크 기능이 분산되어 관리가 어려움
•	나중에 다시 찾기 어려운 구조
•	체계적인 분류 및 태그 기능 부재
•	저장 과정이 복잡하여 탐색 흐름이 끊김
3.2 목표
단기 목표 (3개월): MVP 출시 및 1,000명 사용자 확보
중기 목표 (6개월): 10,000명 사용자, 평균 저장 콘텐츠 50개 이상
장기 목표 (12개월): 100,000명 사용자, 프리미엄 모델 론칭



4. 타겟 사용자
4.1 주요 페르소나
페르소나 1: 취미 수집가 지연 (27세, 직장인)
•	인스타그램, 핀터레스트에서 레시피, 인테리어 아이디어 수집
•	주말마다 저장한 레시피 보며 요리하는 것이 취미
•	각 플랫폼 북마크가 분산되어 찾기 어려움을 겪음
페르소나 2: 크리에이터 민수 (32세, 마케터)
•	유튜브, 트위터에서 마케팅 레퍼런스 수집
•	프로젝트별로 레퍼런스를 체계적으로 정리할 필요
•	빠른 저장과 나중에 분류하는 워크플로우 선호
4.2 사용자 니즈
•	SNS 탐색 중 즉시 저장 (3초 이내)
•	관심사별 컬렉션으로 체계적 정리
•	시각적으로 콘텐츠 빠르게 스캔
•	나중에 쉽게 재발견할 수 있는 검색
•	멀티 디바이스 동기화
 


5. 핵심 기능
5.1 기능 개요
SaveBox는 사용자 맥락에 따라 최적화된 두 가지 UX를 제공합니다:
구분	맥락 1: SNS 빠른 저장	맥락 2: 앱 직접 열기
시나리오	SNS 콘텐츠 탐색 중	저장한 콘텐츠 관리
목표	3초 안에 저장 완료	정리, 탐색, 재발견
UI 타입	토스트 팝업 (최소 UI)	전체 화면 (밀도 높은 UI)

5.2 주요 기능 목록
5.2.1 인증 및 계정
•	소셜 로그인: Google, Apple, Email 지원
•	익명 모드: 로그인 없이 로컬 저장 가능
•	자동 로그인: 최초 1회 인증 후 자동 유지
5.2.2 빠른 저장 (SNS 공유 연동)
•	네이티브 공유 버튼 연동
•	토스트 팝업으로 컬렉션 선택
•	컬러 태그 선택 (선택사항)
•	저장 완료 스낵바 + '보러가기' 버튼
•	자동 썸네일, 제목 추출 (Open Graph)
5.2.3 컬렉션 관리
•	컬렉션 생성: 이름, 컨텐츠 이미지 
•	컬렉션 편집: 이름, 컨텐츠 이미지
•	컬렉션 삭제: 콘텐츠 포함 전체 삭제
•	URL 직접 추가: 복사한 링크를 컬렉션에 추가
•	기본 컬렉션: '전체' 자동 생성
5.2.4 콘텐츠 뷰
•	앨범형 뷰: 2열 그리드, 썸네일 중심
•	리스트형 뷰: 제목, 메타 정보 포함
•	앨범형/리스트형 세그먼트 버튼으로 자유 전환
5.2.5 검색 및 필터
•	제목, 메모로 검색
•	플랫폼별 필터 (Instagram, YouTube 등)
•	최근 저장순 정렬
 


6. 기능 상세 명세
6.1 빠른 저장 플로우
1.	사용자가 Instagram/YouTube 등에서 콘텐츠 발견
2.	'공유' 버튼 클릭 → SaveBox 선택
3.	토스트 팝업 표시 (배경 어둡게 처리)
4.	컬렉션 리스트 표시 (최대 4개 + “새 컬렉션” 만들기)
5.	사용자가 컬렉션 선택
6.	'저장하기' 버튼 클릭
7.	저장 완료 스낵바 표시 ('보러가기' 버튼 포함)
8.	2초 후 자동으로 원래 SNS로 복귀 (또는 '보러가기' 클릭 시 SaveBox 앱으로 이동)
예상 소요 시간: 3-5초 (컬러 선택 포함)

6.2 URL 직접 추가 기능
위치: 새 컬렉션 만들기 화면
기능: 
•	'📋 붙여넣기' 버튼: 클립보드의 URL 자동 감지 및 추가
•	URL 형식 자동 검증(검증완료 시, 우측 Green 체크아이콘)
주의: SNS 빠른 저장에는 이 기능 없음 (불필요)

6.3 저장 후 바로보기
위치: 저장 완료 스낵바 우측
동작: 
•	'보러가기' 버튼 클릭 시: SaveBox 앱으로 전환 → 방금 저장한 콘텐츠 상세 화면
•	무시 시: 2초 후 자동으로 원래 SNS로 복귀
•	상세 화면에서: 메모 추가, 컬러 변경, 원본 링크 열기 등 가능
장점: 저장 직후 즉시 추가 정보 입력 가능

8. 기술 요구사항
8.1 기술 스택
구분	기술
모바일 앱	React Native + Expo (iOS/Android 동시 개발)
백엔드	Firebase / Supabase (인증, DB, 스토리지 통합)
데이터베이스	Firestore / PostgreSQL (실시간 동기화)
인증	Firebase Auth (소셜 로그인 지원)
링크 파싱	Open Graph API (썸네일, 제목 추출)
웹 버전	Next.js + React (Phase 2)

8.2 성능 요구사항
•	저장 완료까지 3초 이내 (90% 케이스)
•	앱 실행 시간 2초 이내
•	검색 결과 응답 1초 이내
•	이미지 로딩 최적화 (Lazy loading)

8.3 보안 요구사항
•	HTTPS 통신 필수
•	사용자 데이터 암호화 저장
•	OAuth 2.0 기반 소셜 로그인
•	개인정보 처리방침, 이용약관 제공

10. 성공 지표
10.1 핵심 지표 (KPI)
•	DAU/MAU: 일간/월간 활성 사용자 수
•	평균 저장 콘텐츠: 사용자당 50개 이상
•	저장 완료율: 90% 이상 (공유 클릭 후 저장 완료)
•	재방문율: 주 3회 이상 (D7 Retention 40% 이상)
•	평균 저장 시간: 3초 이내

10.2 사용자 만족도
•	앱스토어 평점: 4.5점 이상
•	NPS (Net Promoter Score): 50 이상
•	사용자 피드백: 월 1회 설문 및 인터뷰

10.3 비즈니스 지표
•	3개월: 1,000명 사용자
•	6개월: 10,000명 사용자
•	12개월: 100,000명 사용자, 프리미엄 전환율 5%
