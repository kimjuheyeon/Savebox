# 2. 제품 개요

## 2.1 제품 소개

SaveBox는 SNS(Instagram, YouTube, X, Pinterest, TikTok)에서 발견한 콘텐츠를 분산된 북마크에서 벗어나 한 곳에서 통합 저장/관리할 수 있는 모바일 서비스입니다. 특히 공유 흐름 중에 앱 전환 시간을 최소화하는 `맥락 1: SNS 빠른 저장`을 우선 구현합니다.

## 2.2 핵심 가치

- 빠른 저장: SNS 탐색 중 3초 이내 북마크 저장
- 체계적 정리: 컬렉션/컬러 태그 기반 분류
- 쉬운 재발견: 앨범형/리스트형 뷰 및 검색
- 멀티 플랫폼: 주요 SNS 공유 연동

# 3. 제품 목표

## 3.1 단기 우선 목표 (맥락 1 우선)

- 첫 릴리스(Phase 1): 맥락 1 공유 저장 완료률 90% 이상
- 평균 공유 저장 소요 시간 3~5초 달성
- 공유 완료 후 기존 SNS 복귀까지 2초 자동 복귀 동작 안정화

## 3.2 기존 장기 목표(유지)

- 3개월: MVP 출시 및 1,000명 사용자 확보
- 6개월: 10,000명 사용자, 평균 저장 콘텐츠 50개 이상
- 12개월: 100,000명 사용자, 프리미엄 모델 론칭

# 4. 사용자 맥락

본 PRD는 사용자 환경을 2가지 맥락으로 분리합니다.

- 맥락 1: **SNS 빠른 저장(Share Extension)**
- 맥락 2: **앱 직접 열기(Save app)**

현재는 맥락 1을 먼저 완성하고, 맥락 2는 병행 보완 범위로 운영합니다.

# 6. 핵심 사용자 흐름(요구사항)

## 6.1 맥락 1 우선 흐름 (필수) - 구현 완료

### 1. S1-01 공유 대상 선택 진입 화면

- SNS(Threads 등)에서 공유 버튼 탭
- 공유 시트에서 `SaveBox` 앱 탭
- **구현**: iPhone 모바일 프레임(390x844) 내 Threads 스크린샷 배경 + 블랙 오버레이(opacity 0.5)
- 0.8초 후 Share Extension 시트가 하단에서 슬라이드업

### 2. S1-02 콘텐츠 미리보기 카드

- 공유된 콘텐츠의 기본 메타(제목/썸네일) 표시
- **구현**: Horizontal Card UI (배경색 `#F8F8F8`, `rounded-xl`)
- **이미지 있는 경우**: 1:1 썸네일(12x12, `rounded-lg`) + 콘텐츠 타이틀 + 출처
- **이미지 없는 경우**: 타이틀 첫 글자 이니셜(흰색 배경) + 콘텐츠 타이틀 + 출처
- 헤더와 컬렉션 리스트 사이에 배치

### 3. S1-03 컬렉션 리스트 패널

- 기존 컬렉션 목록 표시
- **구현**: 각 컬렉션은 1:1 이미지 썸네일(10x10, `rounded-lg`) + 컬렉션명 + 항목 수 + Plus 아이콘
- 최대 280px 높이, 스크롤 가능
- 기존 컬렉션 탭 시 즉시 해당 컬렉션으로 저장
- 헤더 우측 `새 컬렉션` 버튼으로 S1-04 진입

### 4. S1-04 새 컬렉션 입력

- `새 컬렉션` 탭 시, 컬렉션명 입력 필드 노출 (자동 포커스)
- 공백명 방지 (`disabled` 상태)
- `완료 및 저장` 버튼 탭 시 컬렉션 생성 즉시 반영 후 저장 실행
- `← 뒤로가기`로 S1-03 복귀 가능
- Enter 키로도 생성/저장 가능

### 5. S1-05 저장 완료 스낵바

- 저장 즉시 시트가 슬라이드 다운으로 닫힘
- 하단에 스낵바 토스트 노출
- **구현 스펙**:
  - 배경: `rgba(0, 0, 0, 0.8)` + `backdrop-filter: blur(12px)`
  - 모서리: `rounded-2xl`
  - 좌측: 체크 아이콘(green) + `"컬렉션명"에 저장됨`
  - 우측: `앱 열기` 액션 버튼 (indigo-400)
- 슬라이드업 + 페이드인 애니메이션

### 6. S1-06 자동 복귀

- 스낵바 노출 후 2초 경과 시 자동 dismiss
- 스낵바 사라짐과 함께 블랙 오버레이 해제 → SNS 원래 화면 복귀
- 리셋 버튼으로 전체 플로우 재시작 가능

## 6.2 예외/예외 처리

- 콘텐츠 메타 파싱 실패: 제목/이미지를 최소 폴백 텍스트로 대체 (첫 글자 이니셜), 저장 동작은 여전히 허용
- 컬렉션명이 비정상(공백): `완료 및 저장` 버튼 비활성화로 방지
- 중복 저장 방지: 동일 URL은 중복 탐지 후 `이미 저장됨` 경고와 함께 대체 경로(기존 항목 상세 이동) 제공
- 네트워크 지연: 토스트 로딩 상태 후 3초 내 실패 시 재시도 가이드

# 7. 기능 요구사항 반영(기존 v1.0 + v1.1 + v1.2 반영)

## 7.1 빠른 저장

- `공유` 버튼 → SaveBox 진입
- 콘텐츠 메타 자동 추출(제목/썸네일)
- 콘텐츠 미리보기 카드로 저장 대상 확인
- 기존 컬렉션 바로 저장 (1탭)
- 새 컬렉션 생성 후 저장을 동일 패널에서 처리
- 저장 완료 스낵바(`"컬렉션명"에 저장됨`) 노출
- 보류/미탭시 2초 자동 SNS 복귀

## 7.2 기존 컬렉션 저장

- 목록에서 컬렉션 선택 후 1탭 저장
- 컬렉션별 이미지 썸네일(1:1)로 시각적 구분
- 저장 완료 즉시 시트 닫힘 + 스낵바 출력

## 7.3 새 컬렉션 저장

- 헤더 `새 컬렉션` 버튼 탭 → 컬렉션명 입력
- `완료 및 저장` 탭 시 컬렉션 생성 + 대상 콘텐츠 저장 연동
- 생성된 컬렉션은 목록 최상단에 즉시 반영

# 8. 화면/UX 용어 정합성 규칙

- 저장 피드백은 `스낵바` 또는 `save_success_snackbar`로 통일 (v1.1의 토스트에서 변경)
- `자동 복귀`는 사용자 개입 없이 SNS 원복되는 상태를 지칭
- `앱 열기`는 Save app 진입용 액션(선택)으로 스낵바 우측에 배치

# 9. UI 디자인 스펙 (v1.2 신규)

## 9.1 프로토타입 프레임

- 모바일 프레임: `390px x 844px` (iPhone 14 기준)
- 프레임 스타일: `rounded-[40px]`, `border-[6px] border-neutral-700`
- 배경: `bg-neutral-900` (프레임 외부)

## 9.2 배경 및 오버레이

- 배경 이미지: `public/Image/default-screen.png` (Threads 스크린샷)
- 블랙 오버레이: `bg-black opacity-50` (시트 또는 스낵바 활성 시)

## 9.3 Share Extension 시트

- 배경: `bg-white`, `rounded-t-3xl`
- 그림자: `shadow-[0_-10px_40px_rgba(0,0,0,0.3)]`
- 진입 애니메이션: `translate-y-full → translate-y-0` (300ms ease-out)
- 상단 드래그 인디케이터: `w-12 h-1.5 bg-gray-300 rounded-full`

## 9.4 콘텐츠 미리보기 카드 (S1-02)

- 카드 배경: `#F8F8F8`
- 카드 모서리: `rounded-xl`
- 썸네일: `w-12 h-12`, 1:1 비율, `rounded-lg`
- 이미지 없는 경우: 흰색 배경에 첫 글자 이니셜 표시

## 9.5 컬렉션 리스트 (S1-03)

- 썸네일: `w-10 h-10`, 1:1 비율, `rounded-lg`
- 항목 간격: `p-3`, `rounded-xl`
- 최대 높이: `280px`, 스크롤 가능

## 9.6 스낵바 토스트 (S1-05)

- 배경: `rgba(0, 0, 0, 0.8)` + `backdrop-filter: blur(12px)`
- 모서리: `rounded-2xl`
- 패딩: `px-4 py-3.5`
- 위치: 화면 하단 `bottom-10`
- 좌측: 체크 아이콘(`bg-green-500`, 5x5) + 저장 텍스트(white)
- 우측: `앱 열기` 버튼(`text-indigo-400`)

# 10. 성능/품질 요구사항

- 맥락 1에서 저장 완료까지 3초 이내(90% 케이스)
