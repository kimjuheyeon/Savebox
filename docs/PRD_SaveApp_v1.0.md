## 1.1 문서 범위

본 문서는 SaveBox의 **맥락 2: 앱 직접 열기(Save App)** 환경에서의 기능을 정의합니다.

- 맥락 1(SNS 빠른 저장)은 별도 PRD v1.2 참조
- 맥락 1과 맥락 2는 동일한 데이터 저장소를 공유하되, UX는 독립적으로 설계

# 2. 제품 개요

## 2.1 맥락 2 정의

**맥락 2: 앱 직접 열기**는 사용자가 SNS 공유가 아닌, SaveBox 앱을 직접 실행하여 저장된 콘텐츠를 탐색·관리·검색하는 환경입니다.

## 2.2 핵심 목표

- **효율적 탐색**: 저장된 콘텐츠를 앨범형/리스트형으로 빠르게 브라우징
- **체계적 관리**: 컬렉션 생성/편집/삭제, 컬러 태그 분류
- **강력한 검색**: 제목, 메모, 출처 기반 검색 및 필터링
- **콘텐츠 활용**: 원본 링크 접근, 메모 추가, 공유, 삭제

## 2.3 맥락 1과의 차이점

| 구분 | 맥락 1 (Share Extension) | 맥락 2 (Save App) |
|---|---|---|
| 진입점 | SNS 공유 버튼 | 앱 아이콘 직접 탭 |
| 주요 목적 | 빠른 저장 (3~5초) | 콘텐츠 탐색/관리 |
| UX 우선순위 | 속도 최우선, 최소 단계 | 가독성, 편의성 |
| 화면 구조 | 바텀 시트 (단일 시트) | 전체 화면 네비게이션 |
| 체류 시간 | 5초 이내 | 1분 이상 예상 |

# 3. 사용자 시나리오

## 3.1 주요 사용 케이스

### UC-01: 최근 저장 항목 확인
> "오늘 저장한 레시피 다시 보고 싶어."

1. SaveBox 앱 실행
2. 홈 화면에서 최근 저장 항목 섹션 확인
3. 레시피 카드 탭 → 상세 화면 진입
4. 원본 링크로 이동 또는 메모 확인

### UC-02: 컬렉션별 콘텐츠 탐색
> "지난달 저장한 UI 레퍼런스 모아보기."

1. 홈 화면에서 "컬렉션" 탭 선택
2. "UI 레퍼런스" 컬렉션 탭
3. 앨범형 뷰로 썸네일 탐색
4. 원하는 항목 탭 → 상세 확인

### UC-03: 키워드로 검색
> "저번에 저장한 '파스타' 레시피 찾기."

1. 홈 화면 상단 검색바 탭
2. "파스타" 검색어 입력
3. 제목/메모에서 매칭되는 항목 리스트 표시
4. 항목 선택 → 상세 확인

### UC-04: 컬렉션 생성/편집
> "새로운 '여행지 아이디어' 컬렉션 만들기."

1. 컬렉션 탭 → 우측 상단 "+" 버튼
2. 컬렉션명 입력: "여행지 아이디어"
3. 컬러 태그 선택 (선택사항)
4. 저장 → 컬렉션 목록에 추가됨

### UC-05: 콘텐츠 메모 추가
> "이 레시피에 '다음엔 치즈 더 넣기' 메모 남기기."

1. 콘텐츠 상세 화면 진입
2. 메모 섹션 탭
3. 메모 입력: "다음엔 치즈 더 넣기"
4. 자동 저장 → 메모 반영

### UC-06: 콘텐츠 이동
> "이 항목을 '나중에 볼 영상'에서 '완료한 영상'으로 옮기기."

1. 콘텐츠 상세 화면에서 "컬렉션 변경" 탭
2. 대상 컬렉션 선택: "완료한 영상"
3. 확인 → 콘텐츠가 새 컬렉션으로 이동

## 5.1 A1-01: 홈/대시보드 (home_dashboard)

### 5.1.1 레이아웃

```
┌─────────────────────────────────┐
│  SaveBox          🔍  ⚙️        │ ← 헤더 (검색, 설정)
├─────────────────────────────────┤
│  최근 저장 항목                   │
│  ┌────┐ ┌────┐ ┌────┐          │ ← Horizontal Scroll
│  │img │ │img │ │img │          │
│  └────┘ └────┘ └────┘          │
│                                 │
│  컬렉션                          │
│  ┌──────────────────┐           │
│  │ 📁 UI 레퍼런스    │  32개    │
│  ├──────────────────┤           │
│  │ 📁 개발 아티클    │  18개    │
│  ├──────────────────┤           │
│  │ 📁 맛집 리스트    │  12개    │
│  └──────────────────┘           │
│                                 │
│          [+ 새 컬렉션]          │
└─────────────────────────────────┘
   🏠   📚   ⭐   👤               ← 탭바 네비게이션
```

### 5.1.2 구성 요소

**헤더**
- 좌측: "SaveBox" 로고
- 우측: 검색 아이콘, 설정 아이콘

**최근 저장 항목 섹션**
- 최근 7일 이내 저장된 항목 최대 10개
- Horizontal Scroll 카드 UI
- 카드: 썸네일(1:1, 80x80) + 제목(1줄)
- 탭 시 A1-04 상세 화면 진입

**컬렉션 섹션**
- 전체 컬렉션 리스트 (최대 5개 미리보기)
- 각 항목: 컬렉션 아이콘 + 컬렉션명 + 항목 수
- 탭 시 A1-03 콘텐츠 목록 진입 (해당 컬렉션 필터)
- "더보기" 버튼 → A1-02 컬렉션 관리 이동

**새 컬렉션 버튼**
- 중앙 배치
- 탭 시 컬렉션 생성 모달 표시

**탭바 네비게이션**
- 홈, 컬렉션, 즐겨찾기, 프로필

### 5.1.3 동작 사양

- 초기 로딩: 최근 항목 우선 렌더링 (0.5초 이내)
- 컬렉션 데이터는 비동기 로딩 (1초 이내)
- 빈 상태: 최근 항목 없을 시 "아직 저장된 콘텐츠가 없어요" 안내 + 공유 가이드

## 5.2 A1-02: 컬렉션 관리 (collection_management)

### 5.2.1 레이아웃

```
┌─────────────────────────────────┐
│  ← 컬렉션            [편집] [+] │
├─────────────────────────────────┤
│                                 │
│  ┌──────────────────────────┐  │
│  │ 🖼️ UI 레퍼런스      32개 │→│
│  ├──────────────────────────┤  │
│  │ 📄 개발 아티클      18개 │→│
│  ├──────────────────────────┤  │
│  │ 🍴 맛집 리스트      12개 │→│
│  ├──────────────────────────┤  │
│  │ 🎬 나중에 볼 영상    8개 │→│
│  └──────────────────────────┘  │
│                                 │
│  [+ 새 컬렉션 만들기]           │
│                                 │
└─────────────────────────────────┘
```

### 5.2.2 기능 요구사항

**컬렉션 리스트**
- 전체 컬렉션 표시 (최신순 또는 이름순)
- 각 항목: 썸네일 이미지(40x40) + 컬렉션명 + 항목 수 + 화살표
- 탭 시 A1-03 콘텐츠 목록 진입

**편집 모드**
- 우측 상단 "편집" 버튼 탭
- 각 컬렉션 좌측에 체크박스 또는 핸들 표시
- 삭제, 순서 변경 가능
- "완료" 버튼으로 편집 종료

**새 컬렉션 생성**
- "+" 버튼 또는 하단 "새 컬렉션 만들기" 탭
- 모달 팝업:
  - 컬렉션명 입력 (필수, 최대 30자)
  - 컬러 태그 선택 (선택사항, 8색)
  - 아이콘 선택 (선택사항, 기본 폴더 아이콘)
- "생성" 버튼 → 컬렉션 목록 최상단 추가

**컬렉션 편집**
- 컬렉션 롱 프레스 또는 편집 모드에서 탭
- 모달 팝업: 컬렉션명, 컬러 태그 수정
- "저장" 버튼 → 변경사항 반영

**컬렉션 삭제**
- 편집 모드에서 체크 후 하단 "삭제" 버튼
- 확인 다이얼로그: "컬렉션을 삭제하면 내부 콘텐츠는 '미분류'로 이동됩니다."
- "삭제" 확인 → 컬렉션 제거, 콘텐츠는 보존

### 5.2.3 예외 처리

- 컬렉션명 중복: 경고 메시지 "이미 같은 이름의 컬렉션이 있습니다."
- 빈 컬렉션명: "생성" 버튼 비활성화
- 시스템 컬렉션("모든 항목", "미분류"): 삭제/이름 변경 불가

### 5.3.3 기능 요구사항

**뷰 전환**
- 앨범형 ↔ 리스트형 토글 버튼
- 사용자 선택 기억 (로컬 저장)

**앨범형 뷰**
- 2열 그리드 레이아웃
- 카드: 썸네일(1:1, 전체 너비) + 제목(2줄 말줄임)
- 이미지 없는 경우: 첫 글자 이니셜 + 컬러 배경

**리스트형 뷰**
- 세로 스크롤 리스트
- 항목: 작은 썸네일(60x60) + 제목 + 출처 + 저장 일시
- 우측: 더보기 메뉴(⋮) 버튼

**정렬 옵션**
- 최신순 (기본)
- 오래된순
- 이름순 (가나다/ABC)

**필터 옵션 (메뉴 버튼)**
- SNS 출처: 전체/Instagram/YouTube/X/Pinterest/TikTok
- 컬러 태그: 전체/Red/Orange/Yellow/Green/Blue/Purple/Pink/Gray

**검색**
- 우측 상단 검색 아이콘 → A1-05 검색 화면 이동

**액션**
- 카드/항목 탭 → A1-04 상세 화면
- 더보기 메뉴: 공유, 컬렉션 이동, 삭제

### 5.3.4 성능 요구사항

- 최대 100개 항목까지 일반 렌더링
- 100개 초과 시 가상 스크롤(Virtual Scroll) 적용
- 썸네일 레이지 로딩 (뷰포트 진입 시)

**썸네일 영역**
- 이미지 있는 경우: 16:9 또는 1:1 비율 표시
- 이미지 없는 경우: 첫 글자 이니셜 + 컬러 배경

**메타 정보**
- 제목 (최대 2줄)
- 출처 (Instagram, YouTube 등)
- 컬렉션 태그 (컬러 라벨)

**메모 영역**
- 텍스트 입력 필드 (최대 500자)
- 자동 저장 (1초 디바운스)
- 빈 상태: "메모를 추가하세요" placeholder

**메타 데이터**
- 저장 일시 (YYYY. MM. DD. HH:mm 형식)
- 원본 URL (도메인만 표시)

**액션 버튼**
- "원본 링크 열기": 기본 브라우저로 URL 열기
- "컬렉션 변경": 모달로 컬렉션 선택 → 이동
- "삭제하기": 확인 다이얼로그 → 영구 삭제

### 5.4.3 동작 사양

**메모 자동 저장**
- 입력 후 1초 디바운스
- 저장 성공 시 작은 토스트 "저장됨"

**컬렉션 변경**
- 탭 시 모달 표시
- 전체 컬렉션 리스트 (현재 컬렉션 체크 표시)
- 선택 후 "확인" → 콘텐츠 이동, 상세 화면 유지

**삭제**
- "삭제하기" 탭
- 확인 다이얼로그: "이 콘텐츠를 삭제하시겠습니까?"
- "삭제" 확인 → A1-03 목록으로 복귀

# 6. 데이터 구조

## 6.1 콘텐츠 객체 (Content)

```typescript
interface Content {
  id: string;                    // UUID
  title: string;                 // 콘텐츠 제목
  url: string;                   // 원본 링크
  thumbnailUrl?: string;         // 썸네일 이미지 URL
  source: SNSSource;             // 출처 (Instagram, YouTube, etc.)
  collectionId: string;          // 소속 컬렉션 ID
  colorTag?: ColorTag;           // 컬러 태그 (8색)
  memo?: string;                 // 사용자 메모 (최대 500자)
  createdAt: Date;               // 저장 일시
  updatedAt: Date;               // 마지막 수정 일시
}

type SNSSource = 'Instagram' | 'YouTube' | 'X' | 'Pinterest' | 'TikTok' | 'Other';
type ColorTag = 'Red' | 'Orange' | 'Yellow' | 'Green' | 'Blue' | 'Purple' | 'Pink' | 'Gray';
```

## 6.2 컬렉션 객체 (Collection)

```typescript
interface Collection {
  id: string;                    // UUID
  name: string;                  // 컬렉션명 (최대 30자)
  icon?: string;                 // 아이콘 (emoji 또는 URL)
  colorTag?: ColorTag;           // 컬러 태그
  thumbnailUrl?: string;         // 대표 썸네일 (최근 항목 자동)
  itemCount: number;             // 포함된 콘텐츠 수
  createdAt: Date;               // 생성 일시
  updatedAt: Date;               // 마지막 수정 일시
  isSystem: boolean;             // 시스템 컬렉션 여부 (삭제 불가)
}
```

## 6.3 시스템 컬렉션

| ID | 이름 | 설명 | 삭제 가능 |
|---|---|---|---|
| `all-items` | 모든 항목 | 전체 콘텐츠 표시 | ❌ |
| `uncategorized` | 미분류 | 컬렉션 미지정 항목 | ❌ |
