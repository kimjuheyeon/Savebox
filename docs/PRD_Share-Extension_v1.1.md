# 2. 제품 개요

## 2.1 제품 소개

SaveBox는 SNS(Instagram, YouTube, X, Pinterest, TikTok)에서 발견한 콘텐츠를 분산된 북마크에서 벗어나 한 곳에서 통합 저장/관리할 수 있는 모바일 서비스입니다. 특히 공유 흐름 중에 앱 전환 시간을 최소화하는 `맥락 1: SNS 빠른 저장`을 우선 구현합니다.

## 2.2 핵심 가치

- 빠른 저장: SNS 탐색 중 3초 이내 북마크 저장
- 체계적 정리: 컬렉션/컬러 태그 기반 분류
- 쉬운 재발견: 앨범형/리스트형 뷰 및 검색
- 멀티 플랫폼: 주요 SNS 공유 연동

# 3. 제품 목표

## 3.1 단기 우선 목표 (맥락 1 우선)

- 첫 릴리스(Phase 1): 맥락 1 공유 저장 완료률 90% 이상
- 평균 공유 저장 소요 시간 3~5초 달성
- 공유 완료 후 기존 SNS 복귀까지 2초 자동 복귀 동작 안정화

## 3.2 기존 장기 목표(유지)

- 3개월: MVP 출시 및 1,000명 사용자 확보
- 6개월: 10,000명 사용자, 평균 저장 콘텐츠 50개 이상
- 12개월: 100,000명 사용자, 프리미엄 모델 론칭

# 4. 사용자 맥락

본 PRD는 사용자 환경을 2가지 맥락으로 분리합니다.

- 맥락 1: **SNS 빠른 저장(Share Extension)**
- 맥락 2: **앱 직접 열기(Save app)**

현재는 맥락 1을 먼저 완성하고, 맥락 2는 병행 보완 범위로 운영합니다.

# 5. 화면별 네이밍 (필수 정비)

## 5.1 맥락 1: SNS 빠른 저장(Share Extension)

| 화면 ID | 화면명(한국어) | 영문 네이밍(권장) | 목적 |
|---|---|---|---|
| S1-01 | 공유 대상 선택 진입 화면 | `share_extension_entry` | SNS 공유 시 나오는 대상 목록에서 `SaveBox` 선택 후 진입되는 첫 포인트 |
| S1-02 | 콘텐츠 저장 패널(기본) | `share_save_panel_default` | 공유된 콘텐츠 메타(썸네일/제목/요약) 확인 후 컬렉션 저장 동작 수행 |
| S1-03 | 컬렉션 리스트 패널 | `share_collection_list` | 기존 컬렉션 목록 노출 (최대 4개 노출 + 스크롤/더보기) |
| S1-04 | 새 컬렉션 생성 입력 패널 | `share_create_collection_inline` | 새 컬렉션 명칭 입력 후 즉시 생성 |
| S1-05 | 저장 완료 토스트 | `save_success_toast` | “저장됨” 피드백 + 액션 버튼(저장 상세/닫기) 노출 |
| S1-06 | 자동 복귀 상태 | `auto_return_to_source` | 사용자가 보러가기를 하지 않아도 SNS로 원복되는 상태 처리 |

## 5.2 맥락 2: 앱 직접 열기(Save app)

| 화면 ID | 화면명(한국어) | 영문 네이밍(권장) | 목적 |
|---|---|---|---|
| A1-01 | 홈/대시보드 | `home_dashboard` | 컬렉션, 최근 저장 항목, 검색 진입 제공 |
| A1-02 | 컬렉션 관리 | `collection_management` | 컬렉션 생성/편집/삭제 |
| A1-03 | 콘텐츠 목록 | `content_library` | 앨범형/리스트형 브라우징 및 상세 이동 |
| A1-04 | 콘텐츠 상세 | `content_detail` | 메모, 색상, 원본 링크 확인 및 추가 편집 |

# 6. 핵심 사용자 흐름(요구사항)

## 6.1 맥락 1 우선 흐름 (필수)

1. **S1-01 공유 대상 선택 진입 화면**
   - 트리/스레드에서 공유 버튼 탭
   - 공유 시트에서 `SaveBox` 앱 탭
   - 앱 내부에서는 Share Extension 형태의 최소 뷰로 전환

2. **S1-02 콘텐츠 저장 패널(기본)**
   - 공유된 콘텐츠의 기본 메타(제목/썸네일) 표시
   - 우측/상단에 닫기/취소 제어
   - 기본 액션: 바로 컬렉션 선택 또는 새 컬렉션 생성

3. **S1-03 컬렉션 리스트 패널 (2-A)**
   - 기존 컬렉션 목록 표시
   - `+ 새 컬렉션` 버튼 노출
   - 기존 컬렉션 탭 시 즉시 해당 컬렉션으로 저장(`2-B`)

4. **S1-04 새 컬렉션 입력(2-a)**
   - `+ 새 컬렉션` 탭 시, 컬렉션명 입력 필드 노출
   - 중복명 방지/공백명 방지
   - `완료` 버튼 탭 시 컬렉션 생성 즉시 반영 후 저장 실행(`2-a`)

5. **S1-05 저장 완료 토스트(2-b)**
   - 저장 즉시 `저장됨` 스낵바 노출
   - 액션 버튼: `닫기`(기본), `상세로 이동`(선택)
   - 기본 동작: 5초 내 원래 SNS 컨텐츠 화면으로 자동 복귀

## 6.2 예외/예외 처리

- 콘텐츠 메타 파싱 실패: 제목/이미지를 최소 폴백 텍스트로 대체, 저장 동작은 여전히 허용
- 컬렉션명이 비정상(공백/중복): 에러 문구 후 입력 유지
- 중복 저장 방지: 동일 URL은 중복 탐지 후 `이미 저장됨` 경고와 함께 대체 경로(기존 항목 상세 이동) 제공
- 네트워크 지연: 토스트 로딩 상태 후 3초 내 실패 시 재시도 가이드

# 7. 기능 요구사항 반영(기존 v1.0 + v1.1 반영)

## 7.1 빠른 저장

- `공유` 버튼 → SaveBox 진입
- 콘텐츠 메타 자동 추출(제목/썸네일)
- 기존 컬렉션 바로 저장
- 새 컬렉션 생성 후 저장을 동일 패널에서 처리
- 저장 완료 스낵바(`저장됨`) 노출
- 보류/미탭시 2초 자동 SNS 복귀

## 7.2 기존 컬렉션 저장

- 목록에서 컬렉션 선택 후 1탭 저장
- 저장 완료 즉시 저장된 항목 목록 상단 반영

## 7.3 새 컬렉션 저장

- 새 컬렉션 버튼 탭 → `새 컬렉션명` 입력
- `완료` 탭 시 컬렉션 생성 + 대상 콘텐츠 저장 연동

# 8. 화면/UX 용어 정합성 규칙

- 토스트는 `저장됨 토스트` 또는 `save_success_toast`로 통일
- `자동 복귀`는 사용자 개입 없이 SNS 원복되는 상태를 지칭
- `보러가기`/`상세로 이동`은 Save app 진입용 액션(선택)으로 별도 버튼 라벨링

# 9. 성능/품질 요구사항

- 맥락 1에서 저장 완료까지 3초 이내(90% 케이스)
- 저장 완료 토스트 노출 후 2초 내 자동 복귀
- 컬렉션 생성 응답 1.2초 이내 (평균)
- 저장 실패율 10% 이하
